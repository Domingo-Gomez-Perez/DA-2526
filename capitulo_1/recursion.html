1. Descripción de un problema

El acto de programar es pasar de un modelo descriptivo del problema a un modelo dinámico del problema.

2. Calculando la raíz cuadrada

Calcular la raíz cuadrada de \(\sqrt{x}\) es el número \(y\) tal que \(y^2 = x\).

En general, solamente podemos encontrar una aproximación, por lo tanto nuestra solución devolverá una solución.

3. Pseudocódigo

(define (sqrt-iter guess x)
	(if (good-enough? guess x)
	    guess
	  (sqrt-iter (improved guess x) x)
	  )
	)

4. Procedimientos de caja negra

Un procedimiento caja-negra es aquel cuyo funcionamiento interno no es relevante para quien lo utiliza; solo importa qué hace y no cómo lo hace, esto es:

  1. El nombre del procedimiento.
  2. Los argumentos que recibe.
  3. El resultado que produce.

5. El modelo dinámico

(define (+1p x) (+ x 1)) ;sucesor
(define (-1p x) (- x 1)) ; predecesor

; definición de suma según Dedeking
(define (suma-iter x y)
    (if (= x 0)
	y
	(suma-iter (-1p x) (+1p y))
	)
  )

(define (suma-rec x y)
    (if (= x 0)
	y
	(+1p (suma-rec (-1p x) y))
	)
  )

6. Procedimiento iterativo

1. (suma-iter 1 4)
2. (if (= 1 0) 4 (suma-iter (-1p 1) (+1p 4)))
3. (-1p 1) = 0, (+1p 4) = 5
4. (suma-iter 0 5)
5. (if (= 0 0) 5 (suma-iter ...))
6. 5

7. Procedimiento recursivo

1. (suma-rec 2 5)
2. (if (= 2 0) 5 (+1p (suma-rec (-1p 2) 5)))
3. (+1p (suma-rec 1 5)))
4. (+1p (if (= 1 0) 5 (+1p (suma-rec (-1p 1) 5)))
5. (+1p (if (= 1 0) 5 (+1p (suma-rec (-1p 1) 5)))
6. (+1p (+1p (suma-rec (-1p 1) 5)))
7. (+1p (+1p (suma-rec 0 5)))
...

8. Recursividad vs Iteratividad

  • Un procedimiento recursivo es aquel el proceso generado por la ejecución de este procedimiento mantiene información pendiente en la pila de llamadas.
  • Un procedimiento iterativo es aquel en el que el estado completo del cálculo del proceso se puede describir mediante un conjunto fijo de variables.

9. Resumen

El procedimiento para resolver un problema es un procedimiento recursivo, nos dan un modelo descriptivo, lo pasamos a un código imperativo donde necesitaremos otras funciones de las que tenemos su descripción. Esto acaba, cuando las funciones que tenemos son primitivas.
